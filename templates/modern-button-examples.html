Button Component Examples

Button Types

Text Button

Primary buttons should be used for the main action, secondary for alternative actions, and text buttons for less important actions.

Button Sizes

Choose appropriate button sizes based on importance and context. Large buttons draw more attention, while small buttons work well in compact spaces.

Buttons with Icons

Learn more

Icons can help users quickly understand button actions. Always include accessible labels for icon-only buttons using the aria-label attribute.

Button States

Use button states to provide visual feedback. Disabled buttons should be used when an action is unavailable, and active states show the current selection.

Button Groups

Button groups are useful for related actions or toggling between views. Ensure one option is selected by default when appropriate.

Usage Recommendations

Button Hierarchy

Establish a clear visual hierarchy:

  • Primary buttons: Main action on the page
  • Secondary buttons: Alternative actions
  • Outline buttons: Secondary actions that deserve attention
  • Text buttons: Low-emphasis actions

Accessibility

Ensure buttons meet accessibility standards:

  • Use semantic button elements or ARIA roles
  • Provide adequate contrast (WCAG AA minimum)
  • Include descriptive text or aria-labels
  • Ensure keyboard navigability

Implementation Notes

These button examples use the new design system classes. Make sure to include design-system.css and components.css in your page's <head>.

The button components work with both <button> elements and <a> tags styled as buttons.
